Design and implement a unified "Single API System" for my app (AmyNest AI) using Node.js + Express that serves all app data (Dashboard, Parent Hub, Routine, Amy Coach, Behavior) from one endpoint.

---

🎯 GOAL

- Create ONE central API that returns all required app data
- Eliminate multiple API calls
- Ensure consistent, synced data across all screens
- Improve performance and scalability

---

📡 MAIN API ENDPOINT

Create endpoint:

GET /api/app-data

---

Return structured JSON:

---

🧠 DATA AGGREGATION LOGIC

Inside backend:

- Fetch from DB:
  
  - user profile
  - children data
  - routine data
  - coach progress
  - behavior logs

- Combine all into ONE response

---

📊 CALCULATIONS

1. Routine progress:
   completedCount / totalCount

2. Coach progress:
   currentStep / totalSteps

3. Behavior score:
   average of all metrics

4. Total progress:
   average of routine + coach + behavior

📱 FRONTEND USAGE

In Expo app:

- Call ONLY:
  GET /api/app-data

  - Dashboard
  - Parent Hub
  - Routine screen
  - Amy Coach

🔄 REAL-TIME UPDATE

When user:

- completes routine
- finishes coach step
- updates behavior

→ update DB
→ next API call reflects change

---

- Do NOT duplicate logic in frontend
- Do NOT use static data
