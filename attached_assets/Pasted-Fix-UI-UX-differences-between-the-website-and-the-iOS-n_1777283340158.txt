Fix UI/UX differences between the website and the iOS native (WebView-based) app so that the native app visually and functionally matches the website.

GOAL:
Ensure pixel-level consistency between browser version and iOS app (WebView).

--------------------------------------------------
1. VIEWPORT & SCALING (CRITICAL)
--------------------------------------------------
- Ensure correct meta viewport exists in website:

<meta name="viewport" content="width=device-width, initial-scale=1, maximum-scale=1, user-scalable=no">

- In iOS WebView:
  • disable unwanted zoom
  • ensure content fits screen width

--------------------------------------------------
2. SAFE AREA (iPhone notch fix)
--------------------------------------------------
- Add CSS support:

body {
  padding-top: env(safe-area-inset-top);
  padding-bottom: env(safe-area-inset-bottom);
}

- Ensure no content overlaps notch or bottom bar

--------------------------------------------------

--------------------------------------------------
- Enable:
  • JavaScript
  • DOM storage
  • localStorage / sessionStorage

- Disable:
  • overscroll bounce if breaking UI

- Ensure app always loads updated website
