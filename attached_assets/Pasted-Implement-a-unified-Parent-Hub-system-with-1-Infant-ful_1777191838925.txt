Implement a unified Parent Hub system with:

1) Infant full-access + preview of all higher content
2) Progressive content system for age 2+
3) Preview-gated access for future levels

--------------------------------------------------
1. AGE BAND SYSTEM
--------------------------------------------------
Define age groups:
- 0–2
- 2–4
- 4–6
- 6–8
- 8–10
- 10–12
- 12–15

- Detect currentAgeGroup from child DOB

--------------------------------------------------
2. CONTENT MODEL (PROGRESSIVE)
--------------------------------------------------
Each content must include:
- concept_id (shared across levels)
- age_group
- level
- difficulty
- type

--------------------------------------------------
3. SECTION STRUCTURE (ONLY 2 SECTIONS)
--------------------------------------------------
SECTION 1:
"For {Child Name}"

SECTION 2:
"Explore Next Stage for {Child Name}"

--------------------------------------------------
4. INFANT MODE (0–2 YEARS)
--------------------------------------------------
IF currentAgeGroup == "0–2":

SECTION 1:
- Fetch:
  age_group == "0–2"
- Fully unlocked content

SECTION 2:
- Fetch:
  age_group != "0–2"

- Show ALL higher age content (2–15)

- DO NOT apply concept filtering here
- Show wide variety of content (discovery mode)

--------------------------------------------------
5. INFANT PREVIEW RULES
--------------------------------------------------
- All Explore content = preview-only

On click:
- Allow playback: 5–8 seconds
- Then:
  • Auto stop
  • Show message:

    "This is for when your child grows"
    "You’ll unlock this soon"

- Disable seek beyond preview

--------------------------------------------------
6. STANDARD MODE (AGE 2+)
--------------------------------------------------
IF currentAgeGroup != "0–2":

--------------------------------------------------
SECTION 1: "For {Child Name}"
--------------------------------------------------
- Fetch:
  age_group == currentAgeGroup

- For each concept_id:
  • show ONLY matching level

- Fully unlocked

--------------------------------------------------
SECTION 2: "Explore Next Stage for {Child Name}"
--------------------------------------------------
LOGIC:

Step 1:
- Fetch:
  age_group != currentAgeGroup

Step 2:
- Group by concept_id

Step 3:
- For each concept:
  • select NEXT higher level version (closest age band)
  • optionally include higher levels (secondary)

Step 4:
- Sort:
  • nearest age first

--------------------------------------------------
7. PREVIEW-GATED ACCESS (AGE 2+)
--------------------------------------------------
Explore content:

On click:
- Allow preview: 5–10 seconds
- Then:
  • Auto stop
  • Show overlay:

    "Complete current level to unlock this"

- Disable seek beyond preview

--------------------------------------------------
8. PROGRESSION SYSTEM (AGE 2+)
--------------------------------------------------
Unlock condition:
- 70–80% completion of current age content

On unlock:
- Next age_group content becomes fully accessible
- Moves from Explore → Main (next session)

--------------------------------------------------
9. UI DIFFERENTIATION
--------------------------------------------------
Main Section:
- Normal UI
- Full access

Explore Section:
- Dimmed / premium look
- Lock icon
- Labels:
  • "Next Level"
  • "For Age X+"

--------------------------------------------------
10. MULTI-CHILD SUPPORT
--------------------------------------------------
- Use activeChildId
- Maintain separate:
  • progress
  • unlock status

--------------------------------------------------
11. PERFORMANCE
--------------------------------------------------
- Load main section first
- Lazy load explore section
- Cache grouped data
